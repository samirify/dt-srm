How do I setup my repository

What do you need?

Navigate to config/project-pipelines.yaml and add a new entry under availableProjects with unique code as the key and then Project Properties as described below:

Project properties

Projects’ configuration is saved into the project-pipelines.yaml  file which can be found in the config  folder. All available projects are currently saved under availableProjects  object/array.

Each project consists of a key (unique code)  and four  properties as described below:

  • name : This is a name you choose for your project, and can be anything! We only use it to make it easy for you to distinguish between other projects.
  • files : This is where you will configure and tell the system where to get the files from. Currently it supports BitBucket  version control (vc) and the plan is to expand and support others like SVN, GitHub, … etc.
    To learn how to configure your repository please visit the correspondent help file here:
    - BitBucket:     How do I configure my BitBucket repository?
  • docker : We run each project inside a Docker container so we can build the artifacts almost identical to how you’d build them in your environment. And to do this we’ll need a Docker image and the commands you need to run to build your artifact..
  • remote : This currently supports only FTP but the idea, again, is to expand the support to more protocols like SFTP, Local zip files, Cloud hosting, … etc..

NOTE: Please remember NOT to commit any of your project values back to your repository. Soon I’ll be releasing an update where projects configuration is moved to a yaml file (just like bitbucket-pipelies and hopefully I’ll find the time to secure the app behind some authentication so no need to have config files that hold sensitive information.
