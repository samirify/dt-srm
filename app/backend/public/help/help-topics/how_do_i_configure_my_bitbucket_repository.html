How do I setup my BitBucket repository

What do you need?

Make sure you have a BitBucket account (it’s free 😎). Make sure you’re logged in and that you have at least one repository before proceeding to the steps below.

Once ready, navigate to config/project-pipelines.yaml and under you project > files property populate the properties below.

Project files properties

There are currently 7 properties that you need to provide, as follows:

  • vc:  Of course will always be bitbucket  here 🙂
  • workspace:  This is your BitBucket’s workspace where your repository is hosted.
  • repoName:  This is your repository name.
  • repoToken:  This is an access token which is needed when making requests to BitBucket APIs. You can create one here: https://bitbucket.org/YOUR_WORKSPACE/YOUR_REPO_NAME/admin/access-tokens .
    NOTE:  Make sure to replace YOUR_WORKSPACE  and YOUR_REPO_NAME  with your workspace and repository name, respectively.
  • appUser:  This can be found here https://bitbucket.org/account/settings/ . Under Bitbucket profile settings > Username  
  • appPassword:  You need to create an app in BitBucket if you don’t already have one. Then create a password here: https://bitbucket.org/account/settings/app-passwords/  
  • nestedFolderName:  Sometimes you might have a repository which consists of two apps (i.e. backend and frontend) and you’d want to deploy only one. In this case this property comes in handy as you set it to the nested folder relative to the root.
